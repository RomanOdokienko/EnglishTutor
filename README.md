# English Tutor

English Tutor analyses recorded or uploaded English conversation sessions,
builds reproducible progress indicators and generates a static browser UI.

Development is local-first. The accepted production topology is a static
frontend on Vercel with a separately deployed Python backend that owns session
files, provider credentials and all write operations.

## Quick start

Prerequisite: Python 3.13.

Build saved sessions:

~~~powershell
python cli.py
~~~

Start the local application:

~~~powershell
python server.py
~~~

Then open:

- http://127.0.0.1:8000/web/highlights.html — session insights
- http://127.0.0.1:8000/web/progress.html — canonical progress charts
- http://127.0.0.1:8000/web/upload.html — transcript upload
- http://127.0.0.1:8000/web/record.html — audio recording

## Project data and build output

| Location | Purpose |
| --- | --- |
| sessions/<date>/ | Versioned session inputs: metadata, transcript and optional raw audio |
| data/people.json | Speaker aliases |
| web/ | Editable frontend source |
| out/sessions/<date>/analysis.json | Generated per-session analysis |
| out/history.json | Generated cross-session index |
| out/web/ | Generated static artifact published by Vercel |

The project commits session inputs, analyses and generated artifacts. Edit only
web/ for frontend changes; out/web is regenerated by the build and should never
be hand-edited.

## Optional providers

### OpenAI

Set this variable to enable model analysis, annotations and generated
exercises:

~~~powershell
$env:OPENAI_API_KEY="your-token"
~~~

Useful overrides:

~~~powershell
$env:OPENAI_MODEL="gpt-4o-mini"
$env:OPENAI_ANNOTATION_MODEL="gpt-5-mini"
$env:OPENAI_ANNOTATION_RESUME="1"
$env:OPENAI_ANNOTATION_NO_FALLBACK="1"
$env:OPENAI_CHUNK_METRICS="1"
$env:OPENAI_CHUNK_METRICS_RESUME="1"
~~~

Run a CLI build with model analysis:

~~~powershell
python cli.py --use-openai
~~~

### AssemblyAI

The recording flow uploads raw audio to AssemblyAI for transcription:

~~~powershell
$env:ASSEMBLYAI_API_KEY="your-token"
~~~

ASSEMBLYAI_LANGUAGE optionally selects a transcription language. Never commit
provider tokens.

## Canonical metrics

The progress series uses participants[].derived, versioned by
analysis_version.metrics and analysis_version.taxonomy. Legacy LLM summary
fields remain temporary compatibility data and are not the authoritative trend.

After changing a derived formula or taxonomy:

~~~powershell
python cli.py --recompute-derived
~~~

Commit the regenerated outputs with the related source and documentation change.

## Deployment target

Vercel publishes the static out/web directory. A separately hosted persistent
Python backend runs server.py and stores sessions/, data/ and out/.

At static build time set ENGLISH_TUTOR_API_BASE_URL to the public backend URL.
On the backend set ENGLISH_TUTOR_CORS_ORIGIN to the exact Vercel origin. Keep
OpenAI and AssemblyAI keys only in the backend environment.

The current backend has no application authentication. Do not expose its write
API publicly until access control has been selected and implemented.
